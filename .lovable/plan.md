

# نظام مراقبة حضور وانصراف الموظفين والعاملين

## نظرة عامة

بناء نظام متكامل لتسجيل ومتابعة حضور وانصراف العاملين بالمجمع (موظفين، إداريين، مشرفين، معلمين...) مع جداول دوام مرنة، وحسابات تلقائية للتأخير والخروج المبكر، وتقارير يومية وفردية. يتم ذلك بإضافة جداول وواجهات جديدة فقط دون المساس بأي موديول قائم.

---

## المرحلة الاولى: تعديل قاعدة البيانات

### تعديل جدول profiles (اضافة اعمدة)

| العمود الجديد | النوع | الافتراضي | الوصف |
|--------------|-------|-----------|-------|
| job_title | text | NULL | المسمى الوظيفي |
| department | text | NULL | القسم / الادارة |
| is_staff | boolean | true | هل هو من العاملين بالمجمع |

### جدول جديد: `staff_attendance_shifts`

| العمود | النوع | الوصف |
|--------|-------|-------|
| id | uuid PK | المعرف |
| name | text | اسم الدوام (صباحي، مسائي) |
| start_time | time | وقت البداية |
| end_time | time | وقت النهاية |
| grace_in_minutes | integer (default 10) | دقائق السماح للتأخير |
| grace_out_minutes | integer (default 10) | دقائق السماح للخروج المبكر |
| active | boolean (default true) | فعال |
| created_at | timestamptz | التوقيت |

### جدول جديد: `staff_attendance`

| العمود | النوع | الوصف |
|--------|-------|-------|
| id | uuid PK | المعرف |
| staff_id | uuid FK -> profiles.id | مرجع للعامل |
| attendance_date | date | التاريخ |
| check_in_time | timestamptz | وقت الدخول |
| check_out_time | timestamptz | وقت الخروج |
| status | text | حاضر / متأخر / غائب / خروج_مبكر / اجازة |
| shift_id | uuid FK -> staff_attendance_shifts.id | مرجع للدوام |
| late_minutes | integer (default 0) | دقائق التأخير |
| early_leave_minutes | integer (default 0) | دقائق الخروج المبكر |
| total_work_minutes | integer (default 0) | مجموع دقائق العمل |
| notes | text | ملاحظات |
| created_at, updated_at | timestamptz | توقيتات |

- قيد تفرد: `UNIQUE(staff_id, attendance_date)`

### سياسات RLS

- **المدير**: وصول كامل (CRUD) على الجدولين
- **المشرف / السكرتير / الاداري**: تسجيل الحضور والانصراف + عرض السجلات + التعديل
- **المعلم / المساعد**: عرض سجلهم الشخصي فقط (قراءة)

## المرحلة الثانية: منطق احتساب الحالة

### عند تسجيل الحضور (Check-in)
- مقارنة وقت الدخول مع `start_time` من جدول الدوام
- اذا تجاوز `start_time + grace_in_minutes` -> الحالة = "متأخر" مع حساب `late_minutes`
- والا -> الحالة = "حاضر"

### عند تسجيل الانصراف (Check-out)
- مقارنة وقت الخروج مع `end_time`
- اذا كان قبل `end_time - grace_out_minutes` -> "خروج_مبكر" مع حساب `early_leave_minutes`
- حساب `total_work_minutes` = الفرق بين الدخول والخروج

### الغياب
- عند عرض التقارير: اذا لم يوجد سجل لموظف في تاريخ معين يُعتبر "غائب"

## المرحلة الرابعة: واجهة تسجيل الحضور (`StaffAttendance.tsx`)

### التصميم

```text
+--------------------------------------------------+
|  حضور العاملين          [اختيار التاريخ] [الدوام] |
+--------------------------------------------------+
| الاسم | القسم | المسمى | الدخول | الخروج | الحالة |
|-------|-------|--------|--------|--------|--------|
| أحمد  | إداري | سكرتير | [تسجيل]| [تسجيل]| حاضر  |
| محمد  | تعليم | معلم   | 03:50  | [تسجيل]| متأخر |
| خالد  | أمن   | حارس   |   --   |   --   | غائب  |
+--------------------------------------------------+
```

- اختيار التاريخ (اليوم افتراضيا)
- اختيار الدوام (Shift)
- قائمة العاملين (`profiles` حيث `is_staff = true`)
- لكل عامل:
  - زر "تسجيل حضور" يسجل الوقت الحالي ويحسب الحالة تلقائيا
  - زر "تسجيل انصراف" يسجل الخروج ويحسب ساعات العمل
  - امكانية التعديل اليدوي (للمدير والمشرف)
- عرض ملخص اعلى الصفحة: عدد الحاضرين / المتأخرين / الغائبين

## المرحلة الخامسة: سجل الحضور (`StaffAttendanceLog.tsx`)

### الفلاتر
- التاريخ (من - الى)
- الموظف (قائمة منسدلة)
- القسم
- الحالة

### الاعمدة
- الاسم، القسم، وقت الدخول، وقت الخروج، الحالة، دقائق التأخير، دقائق الخروج المبكر، ساعات العمل

### التقارير
- **تقرير يومي**: بطاقات ملخصة (حاضرون، غائبون، متأخرون، خروج مبكر، مجموع ساعات العمل)
- **تقرير فردي**: اختيار موظف + فترة -> عدد ايام الحضور، الغياب، مجموع التأخير، مجموع الخروج المبكر، مجموع ساعات العمل
